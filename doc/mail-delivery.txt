Mail delivery (development and local testing)

This document explains how to test and configure mail delivery for local development. It covers using MailHog (recommended for dev), how to wire the application to MailHog, basic checks, and notes for using a real SMTP provider (e.g. Outlook) for verification.

1. Overview

- For local development we recommend using MailHog — it captures all outbound mail and provides a web UI and API so you can inspect messages without sending them to external addresses.
- Use MailHog when running functional tests or manual verification.
- For integration tests that need a real SMTP provider, see the Outlook section below. Be aware of security (2FA, app passwords) and network restrictions.

2. MailHog (recommended for dev)

Start MailHog with Docker (recommended):

```bash
# Start MailHog detached; SMTP on 1025, Web UI/API on 8025
docker run --rm -d -p 1025:1025 -p 8025:8025 --name mailhog mailhog/mailhog
```

- Web UI: http://127.0.0.1:8025
- API (messages): http://127.0.0.1:8025/api/v2/messages
- SMTP endpoint (where your app should send mail): 127.0.0.1:1025

3. Configure the application to use MailHog

There are two common ways the server reads SMTP configuration in development:
- Environment variables (preferred for runtime)
- `application.properties` or `gradle.properties` at build/runtime (used in some setups)

Example environment variables (for dev to use MailHog):

```bash
export smtpHost='127.0.0.1'
export smtpPort='1025'
export smtpAuth='false'    # MailHog does not require auth
export smtpStarttlsEnable='false'
```

If your app reads `smtpHost`/`smtpPort` from `application.properties`, ensure those property names match what your application expects. Example snippet for `application.properties` (development profile):

```properties
# dev mail (MailHog)
smtp.host=127.0.0.1
smtp.port=1025
smtp.auth=false
smtp.starttls.enable=false
```

5. Common commands to verify MailHog and SMTP

- Check MailHog API messages (direct curl to localhost — bypass proxies if necessary):

```bash
# Ensure NO_PROXY includes localhost when you're behind a proxy
NO_PROXY=127.0.0.1,localhost curl -s http://127.0.0.1:8025/api/v2/messages | jq '.'
```

- If your environment is using HTTP proxy variables (HTTP_PROXY/HTTPS_PROXY), make sure to bypass them for localhost (set `NO_PROXY=127.0.0.1,localhost`). Many helper scripts in this repo do this automatically.

6. Using Outlook (smtp-mail.outlook.com) — if you need a real SMTP

If you must use Outlook for testing:
- Use host: `smtp-mail.outlook.com`, port `587`, STARTTLS.
- If the account has 2FA, you typically need an app password. If using an Office 365 tenant, SMTP AUTH may be disabled by default for the tenant.

Quick connectivity checks (run locally):

```bash
# Check TCP connect
nc -vz smtp-mail.outlook.com 587

# Check STARTTLS / server certificate
openssl s_client -starttls smtp -connect smtp-mail.outlook.com:587 -crlf
```

Python send sample (local run):

```python
import smtplib, ssl
from email.message import EmailMessage

msg = EmailMessage()
msg.set_content('test body')
msg['Subject'] = 'Test'
msg['From'] = 'you@outlook.com'
msg['To'] = 'you@outlook.com'

with smtplib.SMTP('smtp-mail.outlook.com', 587, timeout=10) as server:
    server.ehlo()
    server.starttls(context=ssl.create_default_context())
    server.ehlo()
    server.login('you@outlook.com', 'APP_PASSWORD')
    server.send_message(msg)
```

Troubleshooting Outlook:
- If login fails: check credentials / app password / tenant-level SMTP AUTH settings.
- If connection refused: network or ISP may block outbound SMTP.
- For Microsoft 365 accounts, verify SMTP AUTH is enabled for the mailbox or use Graph API to send mail.

7. Security and best practices

- Never commit plain text credentials into the repository. Use environment variables, CI secrets, or a secret manager.
- For local dev prefer MailHog or smtp4dev to avoid sending real emails.
- If you must use a real SMTP account for QA, create a dedicated test account and enable appropriate security controls (app password, limited mailbox scope).
